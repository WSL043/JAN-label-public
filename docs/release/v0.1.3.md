# v0.1.3

Desktop-shell UI reset release focused on making `admin-web` behave like a desktop operations console instead of a long web page.

## Added

- Desktop-style application shell with a navigation rail, central workspace, right-side inspector, and bottom status bar.
- Explicit workspace lanes for `Draft`, `Template`, `Queue`, and `Audit`.
- Inspector activity feed that summarizes recent bridge, dispatch, audit, and template actions.
- Persistent status bar summaries for bridge health, draft state, queue state, and proof review state.

## Changed

- Manual draft preview now prioritizes the live payload instead of a stale staged snapshot.
- Compose, template, queue, and audit work are separated into desktop-style operator lanes.
- Queue clearing is handled through a dedicated command instead of inline state mutation.
- The visual language now follows a denser desktop console layout instead of a landing-page-style panel stack.

## Fixed

- Reduced stale preview confusion between saved draft snapshots and the current edited draft.
- Tightened manual submit flow so the currently validated draft is the one being sent.

## Follow-up

- `T-046` remains for queue and audit grid tooling such as sort, filter, and larger-data navigation.
- `T-047` remains for splitting the template lane into clearer authoring sub-modes.
- `T-028f-restore` remains for audit backup restore flow.
