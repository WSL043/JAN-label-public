# v0.1.2

PDF-first patch release focused on local operator flow, proof gating, template parity, and audit operations.

## Added

- Proof inbox plus local proof/dispatch ledgers with approve/reject and pin-to-print flow.
- Local template catalog write-back so template edits can be saved and reused by proof/print dispatch.
- Audit export/retention workflows with scoped JSON export, trim dry-run/apply, backup bundle creation, and backup bundle listing.
- Structured template editor plus Rust renderer preview, backed by template manifest/schema validation.

## Changed

- This patch lane is explicitly PDF-first, centered on deterministic SVG/PDF generation and local audit controls.
- Print gate in `desktop-shell` now requires exact proof match on `templateVersion`, `sku`, `brand`, normalized `jan`, `qty`, and lineage.
- Bridge contracts and catalog resolution were tightened, including packaged vs local template source reporting.
- Legacy proof seed and bridge warning flows are stricter and more explicit.

## Fixed

- Approved proof artifacts are rejected when missing, empty, or not valid PDFs.
- Audit backup naming and listing contracts were aligned.
- Numeric JAN handling and import-side high-risk row detection were hardened.
- Retry and payload validation paths now block stale or misaligned jobs earlier.

## Deferred

- Backup restore remains manual.
- `T-030` GitHub `OPENAI_API_KEY` setup remains deferred for this PDF-only milestone.
- `T-031` physical printer matrix / scan validation remains deferred.
- Local template catalog governance and audit transaction atomicity remain follow-up work.
