# v1.0.0 acceptance

Acceptance matrix for the first formal Windows-only `.NET / WPF` release.

Current batch on `2026-04-19`: `M0 + M1` plus early `M2/M3` Designer/catalog slices, the first local proof-review slice, the next local audit-mirror slice, and the first local proof-create slice: native `Save to Catalog`, native open-from-catalog, and native draft preview, including native preview-refresh from the current Designer surface plus the first hardening pass for catalog path safety, template identity retention, authoring-expression preservation, 12 px/mm geometry round-trip, refresh template retention, debounced preview refresh, one local render-service route for both template-open preview and explicit preview refresh, a shared millimeter-based scene model that now feeds both draft `SVG` and draft `PDF` artifacts for the Designer lane, a native SQLite proof ledger path that now persists WPF approve/reject decisions locally, a native SQLite audit mirror that now syncs live dispatch/audit rows plus bundle metadata before WPF audit export plus the visible `History` / `Print Console` audit rows read back from local SQLite, and a native `Run Proof` path that now writes the proof artifact plus matching proof/dispatch ledger rows locally.

## 1. Release Definition

`v1.0.0` means:

- only `apps/windows-shell` is shipped to operators
- runtime is Windows-only and local-machine only
- output scope is PDF-only
- `apps/admin-web`, `apps/desktop-shell`, companion glue, and superseded legacy app-layer runtime are removed from the release path

## 2. Capability Matrix

| area | required at `v1.0.0` | done when |
| --- | --- | --- |
| Home | native workstation readiness view | workstation state, local storage health, migration state, and workflow blockers are visible in WPF |
| Designer | native authoring + save-to-catalog | operators can open, edit, preview, save, version, and roll back templates without web or companion authority |
| Batch Jobs | native CSV/XLSX import | import, validation, retry, and submit staging all run inside WPF |
| Print Console | native proof + dispatch gate | proof review, lineage checks, artifact checks, and dispatch decisions run inside `.NET` services |
| History | native audit tooling | search, export, retention dry-run/apply, bundle listing, and restore review run inside WPF |
| Render | deterministic `.NET` SVG/PDF | fixture-covered templates render deterministically without Rust preview/render dependency |
| Migration | one-time legacy import | overlay, audit, artifacts, and batch snapshot import once without silent overwrite |
| Packaging | single Windows installer | release ships one WPF installer plus checksum and no legacy runtime companion |

## 3. Automated Gates

Unit coverage must include:

- JAN normalization and checksum validation
- template governance and version rollback
- local catalog path containment and embedded template identity checks
- native authoring expression preservation and geometry round-trip at the canvas scale
- batch retry eligibility
- proof lineage and artifact validation
- audit retention and restore validation
- migration idempotence

Integration coverage must include:

- SQLite bootstrap and transactions
- save-to-catalog persistence
- CSV/XLSX import
- proof approve/reject
- blocked dispatch
- audit export and backup/restore

Golden coverage must include:

- deterministic SVG output
- deterministic PDF output
- exact PDF media-box sizing for the current document scene
- barcode content and placement
- color, border, and layout positioning

## 4. Manual Release Gates

Before tagging `v1.0.0`, complete and record:

- Home lane walkthrough
- Designer authoring and save-to-catalog walkthrough
- Batch Jobs import, validation, retry, and submit walkthrough
- Print Console proof review and gated dispatch walkthrough
- History search, export, retention, and restore walkthrough

## 5. Migration Gates

Required migration checks:

- upgrade workstation with existing local overlay
- upgrade workstation with existing audit/proof state
- migration failure leaves a recoverable state
- repeated startup does not duplicate imported content

## 6. Delete Gates

Legacy code can be removed only when all are true:

- the WPF replacement exists
- the replacement has validation and operator coverage
- handoff and todo docs say the WPF path is authoritative
- release packaging no longer needs the legacy runtime

## 7. Final Release Artifacts

`v1.0.0` release assets must contain:

- one Windows installer for `JanLabel`
- one checksum file for that installer
- release notes
- acceptance evidence from this matrix

`v1.0.0` release assets must not contain:

- `admin-web`
- `desktop-shell.exe`
- companion-only release checks
- legacy dual-app operator entrypoints
