# adr

This directory records durable architecture and operations decisions in time order.

## Rules

- One decision per file.
- Do not rewrite an older ADR to change its meaning.
- Mark each ADR as `Accepted`, `Superseded`, or `Deprecated`.
- When a newer ADR replaces an older one, reference the older ADR from the newer file.
